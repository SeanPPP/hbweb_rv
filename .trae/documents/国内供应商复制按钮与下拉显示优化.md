# 修复 package.json main 字段配置错误

## 问题描述

```
ConfigError: Cannot resolve entry file: The `main` field defined in your `package.json` points to an unresolvable or non-existent path.
```

## 根本原因

* `package.json` 中 `main` 字段设置为 `"index.ts"`

* 项目根目录不存在 `index.ts` 文件

* 该项目使用 `expo-router`，入口点应由 Expo 自动管理

## 解决方案

### 推荐方案：移除 `main` 字段

**理由：**

1. `expo-router` 项目不需要手动指定入口文件
2. Expo CLI 会自动识别 `app/` 目录作为路由起点
3. 避免维护额外的入口文件

**修改内容：**
删除 `package.json` 第 4 行：

```json
"main": "index.ts",
```

### 备选方案（不推荐）

如需显式指定入口，可设置为：

```json
"main": "expo-router/entry"
```

但这在现代 Expo 项目中通常不是必需的。

## 实施步骤

1. ✅ **已分析**：确认问题原因和解决方案
2. **待执行**：修改 `package.json`，移除 `main` 字段
3. **验证**：运行 `expo start` 验证修复

## 预期结果

* 移除 `main` 字段后，Expo CLI 将自动使用 `expo-router` 的默认入口点

* `app/_layout.tsx` 将作为应用根布局

* `app/index.tsx` 将作为首页路由
